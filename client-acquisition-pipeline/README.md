# Overseas Client Acquisition Pipeline

Purpose: build a compliant, repeatable outbound system for selling practical automation and data-cleanup implementation services to overseas small businesses and agencies.

Primary positioning:

> China-based async automation developer helping agencies and small teams with data cleanup, Shopify product workflows, CRM follow-up automation, API scripts, and lightweight AI-assisted prototypes.

## Operating principle

This is not spam blasting.

The workflow is:

1. Find public leads from relevant directories, websites, marketplaces, LinkedIn, and agency portfolios.
2. Score each lead before contacting.
3. Write a short, specific opening message using public signals from the lead's own website.
4. Send from the user's real account only.
5. Paste replies back into Codex for translation, intent analysis, scope clarification, pricing, and follow-up writing.
6. Deliver real work with clear scope, files, notes, and handoff documentation.

## Core customer pools

Start with these three pools:

1. Shopify small agencies
   - Need product data cleanup, CSV import help, bulk editing, supplier catalog cleanup, scripts, ERP/API integration support.
2. Zapier/Make/Airtable automation consultants
   - Need overflow implementation support, webhooks, API scripts, data cleanup, spreadsheet preparation, client handoff docs.
3. Small marketing or website agencies
   - Need form automations, lead routing, CRM cleanup, reporting dashboards, website FAQ prototypes, data scraping with permission.

## Files

- `profile_inputs_needed.md`: what the user should provide before outreach.
- `lead_sourcing_playbook.md`: where and how to find leads.
- `lead_tracker.csv`: working lead database.
- `icp_scorecard.csv`: lead scoring rules.
- `outreach_templates.md`: first-contact and follow-up templates.
- `reply_handling_sop.md`: how to process customer replies with Codex.
- `pricing_and_offers.md`: first offers, price anchors, and package structure.
- `delivery_handoff_sop.md`: how to handle work after a lead becomes a client.
- `daily_execution_plan.md`: daily workflow for 30 days.
- `scripts/generate_outreach_drafts.py`: optional helper to generate draft messages from the CSV.

## First milestone

Do not aim for mass volume first.

First milestone:

- 60 qualified leads in the tracker
- 30 personalized outreach messages sent
- 5 real replies
- 2 discovery conversations
- 1 paid test task

## Compliance rules

- Use real name, real country, and real work examples.
- Do not pretend to be in the US, UK, or another country.
- Do not buy accounts or use fake LinkedIn/Upwork identities.
- Do not scrape private data or bypass platform rules.
- Do not send hundreds of identical emails.
- Do not promise results that depend on the client's sales or market.
- Do not send messages from Codex directly. The user sends them from their own account.
