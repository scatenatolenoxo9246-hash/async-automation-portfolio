# Delivery And Handoff SOP

Use this after a lead becomes a paying client.

## Step 1: Confirm scope in writing

Before starting, confirm:

- Input files or systems
- Expected output
- Deadline
- Price
- Revision limit
- What is not included
- Whether sample data or real data is being used

Template:

```text
To confirm the small test task:

Input: [input]
Output: [output]
Timeline: [timeline]
Price: [price]
Included: [included items]
Not included: [excluded items]
Revision: one round of written feedback

If this looks right, I can start after payment/contract setup and receiving the input files.
```

## Step 2: Create a work folder

Folder structure:

```text
client-project/
  README.md
  input/
  output/
  src/
  notes/
```

## Step 3: Build with auditability

For each project, keep:

- Original input sample
- Transformation rules
- Output file
- Manual review notes
- Known limitations
- Next-step recommendations

## Step 4: Delivery email

```text
Hi [Name],

I finished the first version of the test task.

Included:
- [deliverable 1]
- [deliverable 2]
- [short notes/report]

Important notes:
- [manual review point]
- [assumption]
- [limitation]

Suggested next step:
[specific next step]

Please review the output and send any written feedback. One revision round is included in this test task.
```

## Step 5: Ask for testimonial or next task

After the client accepts:

```text
Glad this was useful.

If you are comfortable with it, I would appreciate a short testimonial or permission to mention this as an anonymized case study.

For the next step, I see two possible improvements:
1. [next task]
2. [next task]
```

## What Codex helps with

Codex can help:

- Translate client requirements into technical tasks
- Draft scope and quote
- Write or review code
- Generate README and handoff notes
- Prepare delivery email
- Create an anonymized case study

User must handle:

- Real account communication
- Payment setup
- Final review before sending
- Client relationship and commitments
